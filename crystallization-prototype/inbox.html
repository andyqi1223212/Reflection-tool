accept / reject 会做什么？（inbox 本身不执行任何命令）
  • accept：只复制一行终端命令。你在仓库根粘贴执行后，orchestrator 仅重跑 push 阶段，按 B 的 output_kind 调 merge --mode new|update|meta，把卡写入 inquiry-chain-demo-v3-good-answer.md 并 export。若你已在 chat 里 merge 过同场景，可能 exit 4 撞 id——应 reject 或删重复段。
  • reject：只复制 mv runs/<id> runs/_rejected/。不删 v3 md、不调 API、不入库；只是把该 run 文件夹挪走，刷新后从 inbox 消失。
  • verdict=pass 但仍在 inbox：通常是 push 失败（如曾经的 meta 未实现）；现在 meta 已支持，accept 一般不需要 --force-pass。
  • conditional_pass / fail：accept 命令会带 --force-pass（人审覆写后 merge）。
