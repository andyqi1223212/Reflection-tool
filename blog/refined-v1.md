# 天天内耗的大学生，用 3 个 agent 给自己造了一个"改命工具"

《了凡四训》里有个改命公式：

> 当前运势 = 初始命数（前因）+ 核心变量（当下的心念与行动）

作为一个天天内耗的大学三年级学生，我意外发现这个公式在 AI 时代是极佳的系统设计指南。

那些被反复触发焦虑、不安全感的具体情景，对应着我根深蒂固的**行为模式**——对评价的恐惧、过度共情、被得失心绑架——也就是我的"初始命数"。而每一次在现实里被触发的瞬间，我脑子里闪过的念头和当下采取的小动作，就是改变局面的"核心变量"。

如果不把这些瞬间沉淀下来，我们就会在同一个坑里反复跌倒。"改命"不需要玄学，只需要看清自己的底层模式，盯紧每一个"核心变量"，靠复利把运势曲线慢慢拉上去。

这就是我做这个工具的起点。

---

场景是真实的刺，痛点在反复重复。

状态不好的时候，我的脑子会突然被这类念头占满：

> 万一没人喜欢我？万一大家都讨厌我？万一我拿不到 offer？努力很多最后什么都没有……

肩膀立刻绷紧，胸口发紧。

我习惯找 AI 深聊——不是问"该怎么办"那种鸡汤套话，而是用第一性原理拆解这些念头背后的底层结构。AI 确实能帮我看清很多自己忽略的东西。

比如有一次，AI 帮我把上面那一串 "what if" 命名成了一个机制：

> what if 不是深度思考，是焦虑借想象力做痛苦制造机。你不是在预测，是在向想象里的"全世界"索要一个永远拿不到的绝对保险。

那一刻我整个人松了——客观现实和主观脑补被彻底剥离开来，紧绷的脑子降了温。这种"被命名"的感觉非常治愈。

---

## 但"懂道理"会过期

跟 AI 聊完的两三天，我整个人通透。再过一周，新情境又把我打回原形。更扎心的是，我想找回之前那段对话，几十个聊天记录翻不出来。

我意识到我**不需要对话全量备份**。我需要的是：被触发的那一秒，能让我在 30 秒内松下来、落到一个小动作上的东西。

我的设计原则只有一句：

> **问题是资产，答案是耗材。**

AI 几千字的共情和建议是耗材——会过期。但我**追问的路径**和**我们一起命名出的那个机制**是资产。我做的卡片只存资产。

我把这种卡片叫**晶体卡**。

举个例子。同一个触发点："万一没人喜欢我、拿不到 offer、最终一事无成"——默认的 ChatGPT 会这样回答：

> 我完全理解你现在的焦虑，这种感受其实很正常……（共情 100 字）你可以试试：1. 写下三件做得好的事 2. 和朋友聊聊 3. 给自己一些时间……记住，你已经很棒了，加油！

读完温暖，五分钟后回到同一个触发点，没记住任何一个字。

下面是我的晶体卡（一张实际产物，结构严格、可机器校验）：

---

**IC-011 · what if 链条**

**机制**

what if 不是深度思考，是焦虑借想象力做痛苦制造机。你不是在预测，是在向世界索要一个永远拿不到的绝对保险——而那群担心你失败的"全世界"，多半是童年观众席投到现在的影子。

**入口句**

> 害怕在心里，我在此刻。

**小动作**

1. 问自己：这个 what if 能导出一个具体动作吗？
2. 能，就写一个降低风险的动作；不能，标记为噪音，不再续航。
3. 回到身体：吸一口气，确认我在此刻，不在那个想象的灾难里。

<details>
<summary>Trigger / 追问路径（折叠：不是常看的，是回溯用的）</summary>

状态不好时，脑中突然入侵的念头：万一没人喜欢我？万一拿不到 offer？what if I try so hard but finally achieve nothing？……

</details>

---

为什么是"机制 + 入口句 + 小动作"这三层，不是别的？

因为被触发的那一秒，人**读不进长文**。我只来得及做三件事：知道**发生了什么**（机制把模糊感受命名）、抓**一句话喘气**（入口句作为短锚）、做**一件最小的、身体能动的事**（小动作把抽象变可执行）。

折叠区里的追问路径平时不看。但下次再陷进去的时候，我可以追溯到这张卡最初是从哪段疼里长出来的。

---

## 难点不在 pipeline 连通，在 agent 协同与语言的一致性

把一段聊天记录变成上面那张卡，技术上不复杂——三个 LLM 调用串起来，谁都会写。但跑过一两个月你会发现，这个工作流真正的瓶颈不在调用链，而在三件 AI builder 容易低估的事。

### 一、多 agent 工作流真正的难点，是把"主观判断"压成"机器契约"

我有三个 agent：A 做心智诊断（从聊天记录里剥出底层模式），B 做风格提炼（把诊断翻成"像我本人说的话"），Judge 做最后的审阅。听起来很标准。

但 Judge 不审"内容对不对"——它审"写作契约有没有破"。

举一条它实际在跑的规则：

> 机制里如果出现命名式刺痛（"你不是 X，是 Y"），小动作里必须包含至少 1 步身体能做的动作，否则打回重写。

为什么？因为刺痛裸奔接不住——你点破了用户某个心态模式，却让他"接纳真实的自己"，他当场没法落地，反而更焦虑。

这条规则我以前是在心里跑的，写卡的时候凭感觉。现在它是 Judge 里一行可机器校验的判断。

**写作品味变成可程序检查的契约**——这才是 multi-agent 真正的工作量，不是 prompt 字数。

### 二、上下文不是越多越好，是越干净越好

我吃过一个亏。

早期我把人工标注过的几百条历史回答全部塞进 B 的 prompt 当示例。结果 B 的输出越来越糖浆化——因为标注里既有"4 分的简洁回答"也有"2 分的婆婆妈妈共情"，AI 取了个平均。

修复方法是收敛**唯一权威文件**：从标注里抽出 8 条核心写作规则、十几句最有冲击力的入口句模板、十几条反例。这一份文件（我把它叫**风格词典**）成为 B 唯一的风格依据，标注全表反而被列入 B 的"禁读清单"。

同样的纪律施加到所有 agent：A 看不到风格词典（避免诊断阶段就开始凹句子）；B 看不到 Judge 的 prompt（避免它学会"应付审查"）；Judge 看不到 A、B 的 prompt（避免评估者被生成者污染）。

一句话：

> **上下文装配（context curation）和 prompt 写作是同等重要的两条腿**——大多数 AI 工作流只练了后面那条。

### 三、风格自进化不用微调，用一份文本权威 + 人在环

最后一个反直觉的选择：我没微调任何模型，也没跑 RLHF。

原因很简单——**风格词典是 markdown 文件，模型权重不是**。markdown 我能用 git 看每次改了什么，能一行回滚，能挑这次合成出来的 10 条规则里哪 3 条合并、哪 5 条拒绝、哪 2 条改完再说。微调一旦合进权重，我失去这些自由。

闭环长这样：

1. 每读一张卡，我在网页上打分 + 写一句吐槽（"这句太爹了，像长辈训话"），存到本地 jsonl。
2. 一个合成 agent 定期跑这些反馈，对比高分句和低分句的特征，写成一批"建议加进风格词典"的提案。
3. 这些提案不会自动生效——它们躺在网页一个**待审收件箱**里，每条都能预览修改前后的 diff。
4. 我点同意，提案合并，风格词典版本号 +1，旧版自动归档；点拒绝，提案进入"已驳回"列表，下次合成时 agent 知道不再提。

整个过程**没有黑盒**：每一条规则进入词典的时间、对应哪条反馈、为什么同意——全部可追溯。

## 收尾

一个月的 dogfood 下来，我攒了 24 张卡。

每当再次被触发、感觉脑子又要紧绷的瞬间——打开主页搜索，看一眼最上方的机制描述、念一遍入口句、做一下第一个小动作。30 秒内人就回到地面。

当我后续发现卡和卡之间有关联，或者对一个老问题又有了新感悟，我会一键打包相关的 3 张历史卡塞进新对话作为上下文，带着复利的深度，开启新一轮和 AI 的深聊。

这是我的"改命工具"，也是我开始学 AI 工程以来，写得最用心的一个项目。
