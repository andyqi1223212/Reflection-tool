# 术语对照表（对外发表用）

整理这篇博客里出现的所有术语取舍。原则：**AI 社群熟的英文保留，自己产品的内部词翻成中文（首次出现时可在括号里附原英文）。**

---

## 保留英文（社群通用，读者一眼能 get）

| 术语 | 说明 |
|---|---|
| `agent` | AI builder 圈内通用 |
| `multi-agent workflow` | 同上 |
| `prompt` | 同上 |
| `fine-tune` / `RLHF` | 反例（"为什么不微调"），社群熟悉 |
| `pipeline` | 同上 |

| `context curation` | 首次出现可加中文括注「上下文装配」 |
| `Judge` | 大写当 agent 名用，社群熟 |
| `markdown` / `git diff` / `git revert` / `jsonl` | 工程通用 |
| `dogfood` | 工程通用 |

## 必须翻译（内部产品词，外人首次见会蒙）

| 我产品里的叫法 | 对外用 | 备注 |
|---|---|---|
| `lexicon` | **风格词典** | 第一次出现可加"（我把它叫风格词典）"，后续都用中文 |
| `anchor` | **入口句** | 中文本身就比英文好读 |
| `pattern` | **心态模式** / **底层模式** | 视上下文 |
| `mechanism` | **机制** | 直接中文 |
| `micro_steps` | **小动作** | 直接中文 |
| `crystallization card` | **晶体卡** | 中文产品名 |
| `schema` | **字段结构** / **结构** | 或不说，靠例子展示 |
| `SSOT` | **唯一权威文件** | 业界其实也用，但中文更亲切 |
| `forbidden_inputs` | **禁读清单** | 中文清楚 |
| `fewshot` | **示例（fewshot）** | 首次括注 |
| `lint` | **格式检查** / **校验** | 或不出现 |

## 灰色地带（看具体读者群再定）

| 术语 | 当前选择 | 备选 |
|---|---|---|
| `IC-011` 这种内部卡 ID | **保留** | 留着反而有"真实产物"的可信度 |
| `withheld 列表` | **改成「已驳回」列表** | 已驳回更口语 |
| `synthesize agent` | **合成 agent** | 或直接说"一个定期跑反馈的脚本" |
