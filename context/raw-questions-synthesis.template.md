# Raw Questions Synthesis · Template

> **角色**：把你历史对话里的关键事实、个人画像、trigger 情境素材压成一份**用户肌理与素材库**，让 Pipeline A 在写新卡时**不必再读全部原始对话** —— 只读 `vision + brief + schema + 现有卡库 + 本文档 + 新 raw question` 就能做出有 insight 的判断。
>
> **本文件是模板骨架**——只展示需要哪些节。Pipeline A 不依赖本模板里的具体内容，所以骨架本身可以非常薄。你 fork 后逐节填入自己的真实素材。
>
> **使用顺序**（来源 Pipeline A 的工作流）：
>
> 1. 来一个新 raw question
> 2. A 先扫 §3 / §4 / §7，判断 routing（new vs update）
> 3. A 复用 §6 的"素材库"作为 anchor / micro_steps 的语料
> 4. 涉及童年 / 家族追溯时翻 §2

## 3. 当前生活坐标（trigger 出现的物理空间）

写新卡或新回答时，AI 应假设用户当下处于以下之一：

- **时间挤压**：（例：秋招倒计时 / 项目 deadline）
- **决策树同时炸开**：（同时在抉择的几条路）
- **节奏**：（你的工作日 / 周末节律）
- **关系状态**：（与家人 / 伴侣 / 同事 / 朋友的当前张力）
- **健康 / 身体**：（睡眠 / 运动 / 慢性问题）
