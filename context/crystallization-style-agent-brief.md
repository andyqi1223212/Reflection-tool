## 1. 沉淀摘要（高密度）

- **你要的第一性原理**：剥离隐含假设 → 少量变量 → 因果链；**不是**用编程术语重写心理学。
- **语气**：偏 **DeepSeek**——口语承接、真实情绪可落地；但**拒绝婆婆妈妈**（反复安慰、空洞夸奖、情感铺陈过长、同一意思换三句话说）。
- **客观层**：吸收 **Gemini 式**分层与锋利推导，但**禁止**高密度 CS/ML/工程隐喻刷屏（算力、编译、CI/CD、root…）；除非用户**主动**用工程语言，否则 metaphor 用生活/身体/关系。
- **Subtle**：冷热短打——**口语承接 + 书面拆解**；一段只推进一个支点（假设 / 边界 / 因果）；一句权限入口收尾。
- **口诀**：口语承接，书面拆解；一句人情，一段因果；拒绝糖浆铺底，拒绝术语阅兵。
- **Crystallization 形状**（与 vision 一致）：机制 1–2 句 + ≤20 字入口句 + ≤3 步小动作；trigger 时只看晶体即可着陆。
- **素材库**：历史良质片段索引见 `回答版本explore/良质回答标注册.md`（A/B/C/D 第一波，**F01–F12、G01** 第二波，**H01–H11** 第三波；**实证标注汇总见该文档 §八**）。
- **一句话总结（实证后）**：**好晶体 = 准机制 + 生活载体 + 短入口**；公式/工程壳是**选配**；长叙事只给第一次，索引层要 **H 级压缩**。

## 2. 用户原文反馈归档（完整保留）

以下为对话中用户 **原话或紧贴原文的引用**，按时间顺序；后续迭代以此为准绳。

### 2.1 探索与标注任务（起始）

用户目标包含：找到「说不上来但是喜欢的」**语言风格**与**回答逻辑**背后的共性；在 trigger / 追问情境里辨认**良质答案**；输出可标注的 md；通过认同点、聊天结束前回答等**迹象**推测质量。

### 2.2 标注后对 DS / Gemini 与「第一性原理」

> 我可能更喜欢 **ds 的回答语气风格**，然后我其实 **不太喜欢 gemini 强行联系编程知识那种**，我只是想让他 **第一性原理分析**，详细分析一下我说可以的语言风格是那样的。

### 2.3 第二波提取验证

> 根据你总结的这些规则，**再给我来一波提取回答**，我来验证。

### 2.4 Subtle：DS × 客观 × 不要絮叨

> 我又标注了，我觉得我喜欢就是那种 **ds 口气**，但也不能太 **婆婆妈妈**那种，结合一些 **第一性原理的那种客观 Gemini**那种，就很难找到那种 **subtle**的感觉，你可以继续详细分析。

### 2.5 本文档任务（合并沉淀）

> 把我们的收获，尤其是我的反馈，迭代，每次分析得到的结果总结成**新一个文档**，标准是可以供 **其他 agent 复用**，表达 **清晰**完整信息的同时减少废话，节约 context。注意我 **original 的 feedback 需要详细保留**。

### 2.6 标注册上的实证备注（摘录，完整格子见 `良质回答标注册.md`）

- **机制认同 ≠ 载体认同**：例如 **C14**——「道理不错、mental model 很好，但不喜欢这种语言风格」；**C03**——「道理不错，不喜欢『脏数据』这个词」。→ **骨架可入库，默认输出改写成生活/身体句**；忌讳词按用户备注替换。
- **长叙事折旧**：**F08、F09**——「太麻烦了」「道理可能还凑活」；同类主题的加长 DS 链在 trigger 场景 **性价比低**。→ **优先 H 系列级压缩句**，同一 pattern **第一条叙事完整、后续只保留命名 + 一句 permission**。
- **公式类条目偏低**：**A03、A04**（愧疚定律、决策公式）认同度 **2**；**高分倾向短锚与能动句**：**B10、B13、C11、C12** 为 **4**。→ **第一性原理 ≠ 万事写成公式**；默认交付 **≤20 字入口 + 机制一句**；公式可作 **折叠附录** 给偏结构化思维的时刻。
- **CS 密度惩罚重复验证**：**C05、C06、C15** 偏低；**C08、C13**「忘了」→ 工程类比 **不利于记忆与情感着陆**，除非用户主动用工程话发起。
- **F 系列验证**：**F01、F02、F05、F07、F11**「不错」；**F06** 认可心理税/道德考试 framing（表中写「自傲不错」，疑为 **自个/自我** 笔误，以标注册原文为准）。**F12**「还行」。

## 4. Agent 执行清单（写回答 / 写 Crystallization 时）

1. **先拆假设**：本段隐含前提是什么？删掉多余前提后剩几条变量？  
2. **机制先于道德**：命名模式（好人牢笼、时间旅行、超前预警等）→ 再谈自责是否 proportionate。  
3. **隐喻纪律**：默认用生活/身体/关系；**一道菜一个隐喻**；编程隐喻仅用户先开口时用。  
4. **段落纪律**：**一段一个支点**；禁止空洞夸奖循环与同义安慰复读。  
5. **锋利协作**：可刺痛，但必须解释「你怎么把中性分析听成否定」。  
6. **收束**：≤20 字可默念句 + 可选 ≤3 步小动作。  
7. **篇幅**：宁短勿灌；情绪完整靠「判断准」不是靠形容词多。

## 6. Pattern tags（索引用，勿当死板分类）

`P-EVAL` `P-OVER` `P-SPIRAL` `P-EFF` `P-UNDER` `P-EXIST` `P-KNOW-DO` `P-FAMILY` — 详见 `inquiry-compound-vision` 与用户词表；同一 tag 可挂多类表面情境。

**标注后出现两条高认同主轴（便于 routing）**：

- **判决外包 / 愧疚链**（F01、A05、H01、**D01** 等）：猜别人的心、有限责任、注意力边界、幽灵债务/好人牢笼 —— 偏 `P-OVER` `P-EVAL` `P-SPIRAL`。
- **自我架构 / 注意力经济**（**B10、B13、C11、C12** 四分档）：Inner Game、存在短锚、创造非发现、得失心偷算力 —— 偏 `P-EFF` `P-EXIST` `P-KNOW-DO`。

Trigger 时先分辨当下痛的是 **「羞/愧/怕别人」** 还是 **「空/漂/偷算力/证明」**，再选主轴晶体，避免总往愧疚库堆。

## 8. 实证归纳：写 Crystallization 时必须叠用的规则（2026-05-08）

以下为标注册打分与备注的**合并版**，供 agent **高效复用**，避免只读 §〇 而忽略用户已验证的边界。

### 8.1 双层过滤器（机制 × 载体）

1. **先判断机制是否值得保留**（因果是否准、是否可命名）。  
2. **再选载体**：默认 **生活/身体/关系**；用户明确反感某词（如「脏数据」）→ **换喻不改机制**。  
3. **Gemini 高密度句**：按 **C14 模式**处理——**mental model 进库，默认 UI 文本重写**，工程类比仅在用户先开口或「高密度版」槽位展开。

### 8.2 公式与短锚的分工

- **偏低**：把愧疚/决策写成 **显式公式**（A03、A04）。  
- **偏高**：极短 **可默念句**、**能动与注意力**（B13、C12、C11、B10）。  
**做法**：每条 IC 的默认面是 **机制 1–2 句 + ≤20 字入口**；公式放入 **optional_expand** 或开发者注释，不要塞在 trigger 首屏。

### 8.3 叙事长度与「折旧」

- 同一底层 pattern（underdog、掏卡、饿久了）**首次**长叙事可能有冲击力；**复述一长串**易「太麻烦」（F08、F09）。  
**做法**：**索引层只放压缩句（H 系列、F01/F05）**；故事放进折叠追问，且同一用户会话内 **避免重复全长展开**。

### 8.4 编程隐喻的硬边界

- 低分集中：**过拟合、算力、ETL、API 软弱**（C05、C06、C15）。  
**做法**：与 §4 第 3 条一致——**默认禁用**；用户消息含代码/系统隐喻时可 **跟进 1 个**同源比喻收尾。

### 8.5 刺痛与着陆配对

- **H11** 类高刺痛自省必须与 **H09、F05** 类「着陆垫」同包：**先机制、再一句权限**，避免单行刺痛裸奔。

### 8.6 待补标与误读提醒

- **G01**、**H01–H11** 尚无数值/验证列，**不得**推定为否定。  
- **C08、C13** 标「忘了」：更可能 **曝光不足或载体干扰**，不是机制必然错。  
- 下一轮优先：**给 H 系列打 1–5**、补 **G01**、试用 **C10（3.7）+ B13/C12** 合并成「得失心 / 异步结算」晶体。
