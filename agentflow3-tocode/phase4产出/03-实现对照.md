# 实现对照（plan §0 + §4 + §5）

## §0 数据流

```
runs/*/manifest.json (status=awaiting_human)
        ↓ runs/_index.py（人工或 CI 触发）
runs/_index.js  →  window.RUN_INDEX
        ↓ <script>（file:// 同步加载）
inbox.js  →  列表 / 过滤 / modal
        ↓ 用户复制
python -m agents_runtime.orchestrate --resume … --from push --force-pass
mv runs/<id> runs/_rejected/<id>
```

## §5.1 `_index.py` 相对 plan 草图的增强

1. `question_basename`：列表标题用文件名而非全路径。
2. `is_error`：`push_result.status ∈ ERROR_PUSH_STATUSES` 或 `last_error`。
3. 无 `a_summary` 时读 `a.json` 抽 `route` / `axis` / `target_ic_id`。
4. `SCORE_KEYS` 固定六维，与 Judge schema 对齐。

## §5.3 inbox.js 要点

| 能力 | 实现 |
|------|------|
| 过滤 | `filterMatches` + chip `data-filter` |
| 相对时间 | `formatRelative`；`<time title=ISO>` |
| 复制 | `navigator.clipboard` + `execCommand` 回退 |
| XSS | `escapeHtml` 渲染用户/模型文本 |
| file 链接 | 按路径段 `encodeURIComponent` |
